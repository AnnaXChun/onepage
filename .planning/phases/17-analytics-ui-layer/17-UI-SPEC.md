## Component Inventory

### LineChart (new for Phase 17)

**Purpose:** Display daily page view trends with period toggle

**Visual spec:**
- Container: `bg-surface rounded-2xl p-6`
- Chart height: `h-[300px]` (desktop), `h-[240px]` (mobile)
- Line color: `oklch(70% 0.14 50)` (accent), stroke-width 2
- Area fill: `oklch(70% 0.14 50)` at 0.15 opacity
- Grid: horizontal lines only, `oklch(30% 0.01 260)` at 0.3 opacity
- X-axis ticks: dates formatted "MMM D" (Mar 22), `text-text-muted`
- Y-axis ticks: abbreviated numbers (0, 1k, 2k), `text-text-muted`
- Tooltip: dark surface bg, white text, date + page view count

**States:**
- Default: Render line chart with area fill
- Loading: Skeleton rectangle `h-[300px] bg-surface animate-pulse rounded-2xl`
- Empty (no data): Message "No page view data for this period" centered in chart area
- Error: Error message with retry, same as stat card error state

**Interaction:**
- Period toggle re-renders chart with new data (no page reload)
- Smooth transition: 300ms ease-out-quart fade when switching periods
- Hover tooltip shows exact date and count

---

### PieChart (new for Phase 17)

**Purpose:** Display referral source breakdown with percentages

**Visual spec:**
- Container: `bg-surface rounded-2xl p-6`
- Chart size: `h-[300px]` (desktop), `h-[240px]` (mobile)
- Pie outer radius: 100 (desktop), 80 (mobile)
- Pie inner radius: 0 (donut style if space allows, otherwise solid pie)
- Labels: percentage on each segment, `text-sm font-medium`, positioned outside
- Legend: horizontal wrap below chart, `flex flex-wrap gap-4 justify-center`

**Segment order:** Direct, Search Engine, Social, Referral, Other

**States:**
- Default: Render pie with colored segments and legend
- Loading: Skeleton circle `w-[200px] h-[200px] mx-auto bg-surface animate-pulse rounded-full`
- Empty: Message "No traffic source data" centered in chart area
- Error: Error message with retry

**Interaction:**
- Hover segment: slight scale (1.02) with 150ms transition
- Tooltip on segment hover: source name + count + percentage

## Interaction Patterns

### Dashboard Load (extended from Phase 11)
1. Show loading skeletons (StatCards + ChartCard wrappers)
2. Fetch analytics data from API
3. Fade in stats with slide-up (300ms ease-out-quart)
4. Fade in charts (300ms ease-out-quart, staggered 100ms after stats)

### Period Switch (for charts)
1. User clicks period option in PeriodSelector
2. Charts show fade transition (150ms opacity 0.5 to 1)
3. Chart data updates with new aggregated values
4. Axes rescale as needed with smooth transition

### Chart Hover
1. Hover over line/area: tooltip appears with date and value
2. Hover over pie segment: segment slightly enlarges, tooltip shows details
3. Mouse leave: tooltip disappears with 150ms fade

### Error Handling
1. If chart data fetch fails: show error state in chart container
2. Error message: "Failed to load chart data. [Retry button]"
3. Retry button re-fetches and re-renders chart

## API Data Contract

**Chart data source:** `getBlogAnalytics(blogId: number, period: string)` from `frontend/src/services/api.ts`

**dailyStats structure (for LineChart):**
```typescript
interface DailyStat {
  date: string;        // ISO date "2026-03-22"
  pageViews: number;
  uniqueVisitors: number;
}
```

**refererSources structure (for PieChart, from Phase 16 API):**
```typescript
interface RefererSource {
  source: 'Direct' | 'Search Engine' | 'Social' | 'Referral' | 'Other';
  pageViews: number;
  percentage: number;  // 0-100
}
```

**Combined chart data interface:**
```typescript
interface ChartData {
  blogId: number;
  blogTitle: string;
  dailyStats: DailyStat[];
  refererSources: RefererSource[];
}
```
