# Milestone v1.7: User Profiles

**Status:** ✅ SHIPPED 2026-03-23
**Phases:** 20-22
**Total Plans:** 6

## Overview

User profile pages with public display, profile editing, and integration with navigation. Includes avatar, bio, social links, VIP badge, visitor counts, and featured site pinning.

## Phases

### Phase 20: Public Profile Display

**Goal:** Public profile page at `/user/{username}` is fully functional with avatar, username, bio, social links, and published sites grid.

**Depends on:** Phase 19 (previous milestone)

**Requirements:** PROF-01, PROF-02, PROF-03, PROF-04, PROF-10

**Plans:** 1/1 plans

- [x] 20-01-PLAN.md - Backend API and frontend profile page with avatar, bio, social links, VIP badge, and published sites grid

**Details:**
- ProfileDTO.java - Public profile response DTO with BlogSummary nested class
- UserProfileController.java - GET `/api/user/profile/{username}` public endpoint (no auth required)
- UserService.java - `getPublicProfile()` method aggregating user data with published blogs
- BlogService.java - `getPublishedBlogsByUserId()` method returning published blogs list
- Frontend: Profile.tsx, ProfileHeader.tsx, BlogGrid.tsx, profileApi.ts

---

### Phase 21: Profile Editing

**Goal:** Authenticated users can edit their profile bio, avatar, and social links in account settings.

**Depends on:** Phase 20

**Requirements:** PROF-05, PROF-06, PROF-07, PROF-08

**Plans:** 1/1 plans

- [x] 21-01-PLAN.md - Backend API and frontend profile editing with bio, avatar, social links, preview link

**Details:**
- UpdateProfileRequest.java - DTO with validation annotations
- PUT /api/user/profile - Requires authentication via JwtUserPrincipal
- UserService.updateProfile() - Sanitizes all inputs to prevent XSS
- AccountSettings.tsx - Tab-based UI with avatar upload, bio textarea, social links

---

---

## Milestone Summary

**Key Decisions:**
- Public profile endpoint does not require authentication
- Social links use inline SVG icons to avoid bundle cost
- Only one blog can be featured per user at a time (clear others before setting new)

**Technical Debt:**
- Database schema additions required: blog_daily_stats table, users.profile columns, blogs.seo/featured columns
